WHAT IS DATA COMMANDR

Data Commandr is a tool for data integration, transformation, analysis as well as other data manipulation and data wrangling tasks.

Data Commandr was created with a single mission: radically simplify and democratize all kinds of operations with data for non-IT users.

Data Commandr employs the familiar spreadsheet approach but instead of cell-formulas it uses column-formulas by avoiding set-oriented operations like join or group-by. Therefore, Data Commandr can be viewed as an "Excel for Big Data" or "Big Data Calculator".

HOW DATA COMMANDR WORKS (USP)

Data Commandr is based on a novel technology for data management and analytics collectively referred to as concept-orientation. It was developed by Dr. Alexandr Savinov and published in more than 20 academic papers: http://conceptoriented.org.

Data Commandr gets rid of set-oriented operations which are know to be difficult to understand and use especially for non-IT users. Instead, it employs the spreadsheet (functional) approach where data elements are defined in terms of other data elements. Yet, in contrast to classical cell-oriented spreadsheet applications, Data Commandr is a column-oriented approach where the user works with columns rather than cells. In other words, the user defines new data columns in terms of the existing data columns by producing a graph of column operations as opposed to a graph of cell operations or a graph of set operations. In this sense, Data Commandr could be characterized as Excel for column-based analytical data integration and transformation.

FOR WHOM DATA COMMANDR IS USEFUL (Customers)

Data Commandr is intended for a broad range of non-IT users starting from data enthusiasts, casual users or business users, and ending with data artisans and analysts. All these users do not possess deep knowledge in mathematics and statistics but need some simple yet powerful tool to solve a problem or answer a question by analyzing available data.

WHAT PROBLEMS DATA COMMANDR SOLVES (Pains and Gains)

Data integration, transformation and other data wrangling tasks account for a great deal or even most of the difficulties in data analysis and constitute the most tedious part of the overall analysis process. These problems are normally solved by experts by means of classical BI or scripting tools. This traditional approach is not only expensive and slow but also error-prone because frequently IT experts do not understand the needs of domain-experts. Also, data wrangling problems could be solved by using spreadsheet applications which however have significant limitations and are also difficult to use in the case of multi-table analysis.

By making data manipulations easier for non-IT users, Data Commandr dramatically shortens the total time for data integration, transformation and analysis.

  • Ease of use: Data Commandr makes complex analytical data integration possible for non-IT users.
  • Fast time-to-value: Data Commandr dramatically shortens the total time for data integration, transformation and analysis from months to minutes.
  • Lower TCO: Data Commandr transfers the burden of authoring BI contents to the end users who know much better what needs to be done.
VERSION HISTORY

Version 0.3.0 (xx.12.2016) Up-to-date status for columns.

Version 0.2.0 (10.12.2016) This version has all three types of columns: calculated (for primitive data types), accumulation (for aggregated columns) and tuple columns (for complex data types). The design has been completely reworked.

Version 0.1.0 (04.09.2016) It is the first version developed as a web application rather than a Windows application. This version relies on a completely new implementation of the data processing backend (Data Commandr Engine).
