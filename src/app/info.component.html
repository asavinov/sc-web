WHAT IS DATA COMMANDR

Data Commandr is a tool for data integration, transformation, analysis and other data manipulation and data wrangling tasks.
Data Commandr was created with a single mission: radically simplify and democratize all kinds of operations with data for non-IT users.
Data Commandr employs the familiar spreadsheet approach but instead of cell-formulas it uses column-formulas, and it avoids set-oriented operations like join or group-by. Therefore, Data Commandr can be viewed as an "Excel for Big Data" or "Big Data Calculator".

FOR WHOM DATA COMMANDR IS USEFUL (Customers)?

Data Commandr is a self-service tool for mass market. It is intended for a broad range of non-IT users starting from data enthusiasts, casual users or business users, and ending with data artisans and analysts. All these users do not possess deep knowledge in mathematics and statistics but need some simple yet powerful tool to solve a problem or answer a question by analyzing available data. The buyers of the product are companies which want to reduce their IT costs by getting analytical tools suitable for business users.

WHAT PROBLEMS DATA COMMANDR SOLVES (Pains)?

Data integration, transformation and other data wrangling tasks account for a great deal or even most of the difficulties in data analysis and constitute the most tedious part of the overall analysis process. These problems are normally solved by experts by means of classical BI tools or scripting tools. This traditional approach is not only expensive and slow but also error-prone because frequently IT experts do not understand the needs of domain-experts.

HOW DATA COMMANDR WORKS AND COMPETITIVE ADVANTAGES (USP)

Data Commandr is based on a novel technology for data management and analytics collectively referred to as concept-orientation. It was developed by Dr. Alexandr Savinov and published in more than 20 academic papers: http://conceptoriented.org.
Data Commandr is as powerful as contemporary query languages (e.g., SQL) but at the same time it gets rid of set-oriented operations which are know to be difficult to understand and use for non-IT users. Rather, it employs the spreadsheet (functional) approach where data elements are defined in terms of other data elements. Yet, Data Commandr is not a spreadsheet because it uses data columns instead of data cells. In other words, the user defines new data columns based on the existing data columns by producing a graph of column operations as opposed to a graph of cell operations or a graph of set operations. In this sense, Data Commandr could be characterized as Excel for column-based analytical data integration and transformation.
