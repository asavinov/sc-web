WORKING WITH TABLES
In Data Commandr, data is organized into many tables which are displayed in the TABLES panel as a list. Each table must have a unique name. However, tables do not have their own defintions, that is, new tables do not result from operations with other tables like in the conventional (set-oriented) approaches.
WORKING WITH COLUMNS

All columns of the selected tables are displayed in the COLUMNS panel. Each column must have a unique name within its table. A column is characterized by its type. Column types is either a primitive type or another column. Real data values that a column stores can be either uploaded or computed from other columns.

COLUMN FORMULAS

Data Commandr uses a column-oriented approach to data representation and processing. This means that a new column can be defined in terms of other columns and hence data a column stores is derived from the data in other columns.

Column formulas are a primary element of Data Commandr because they determine how the data will be processed. Column formulas in this sense are analogous to traditional data queries. Column type determines how column values are computed. Currently there exist four column types:

  • User columns get their data values from outside, for example, using import from CSV
  • Calculated columns compute a primitive value from other values in the current row or values in linked row using an expression.
  • Accumulated (or aggregated) columns are intended for data aggregation and they compute their value from many rows of another table using an accumulate expression.
  • Link columns has a value which references a row in another table (which is its data type).

Calculated Columns

Primitive column formula is an expression which is composed of other column names and operations. Column names used in formulas must be written in square brackets. For example, if we want to compute the sum of two existing columns then the (third) column formula has to be defined as follows: [Column 1] + [Column 2]

Column formulas can use the following operators: addition 2+2, subtraction 2-2, multiplication 2*2, division 2/2, exponential 2^2, unary minus or plus +2-(-2), modulo 2%2.

Column formulas can use the following functions: abs, acos, asin, atan, cbrt, ceil, cos, cosh, exp, floor, log, log2, log10, sin, sinh, sqrt, tan, tanh, signum.

Accumulated Columns

Accumulated formulas use the main expression to initialize its values precisely as it is done for primitive expressions. If the main expression is empty then a default value will be used for initialization.

In addition, this initial value is updated by specifying three additional parameters.

  • Accumulate table It is necessary to specify another table the rows of which will be accumulated.
  • Accumulate path A column path has to be specified that leads from the chosen accumulate table to this table.
  • Accumulate formula This formula is written in terms of the accumulate table. Its purpose is to update this column using values from the accumulate table. This formula has to use an additional parameter [out] which is equal to the current value of the column.

Link Columns

Link formulas have special syntax. They are enclosed in curly brackets and store a comma separated list of assignments: { [Col1] = PRIMITIVE EXPRESSION 1, [Col1] = PRIMITIVE EXPRESSION 2, ... }. Each element of this list is an assignment consisting of two expressions. The left hand part is some column of the type table of this column. The right hand part is an expression written using columns of this table.

EVALUATING AND DISPLAYING DATA
When a formula is defined or modified then it is not immediately evaluated and the column data is not updated. To evaluate formulas it is necessary to press the Evaluate button and then the (up-to-date) data of the currently selected table has to be loaded by pressing the Load button. The result is then shown as a number of rows where values have been either previously uploaded or computed from other columns.
UPLOADING DATA
Columns whithout a formula get their data from outside by uploading it to the selected table. Currently Data Commandr supports import from CSV format. This means that the first row of the text has to be a list of column names, for example, Column 1, Column 2. All other rows store values for these columns, for example, the second row could contain the following two values: 22.22, 33.33.
